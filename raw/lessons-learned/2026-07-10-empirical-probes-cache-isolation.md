# Session Reflection: Empirical probes, entity-keyed cache isolation, live-file tailing

**Date**: 2026-07-10
**Session Goal**: Consolidate two diverged copies of a data-analysis library, then iteratively harden it — mapping a running session to its OS process, redesigning a cache for isolation, adding liveness + incremental tailing for live files, and moving a filter to the right layer.

---

## What Went Well

- **Live empirical probing over abstract reasoning.** For a "how is a session mapped to its running process" question, I *ran* the tools (`lsof`, `ps`, `env`, a parent-process walk) instead of theorizing. That surfaced that the runtime injects identity into every child process's environment — turning an "unsolvable from the outside" problem (the working file isn't held open, the id isn't in the process argv, cwd/mtime are racy) into a trivial "read an env var + walk parents to the known binary." Every claim was tied to a concrete command output.
- **Reproduce-before-diagnose.** When told something "broke," I ran the exact failing code and found the cause was *temporal* — a record the code auto-selected had aged out of a fixed time window — not a recent change of mine. Avoided a wrong self-attribution.
- **Confidence calibration.** I reported one defect as *reproduced* (ran it repeatedly and watched the output flap) and a related one as a *design-level inference* (never actually triggered at the scale that would expose it), and said which was which.
- **Bidirectional-merge awareness.** The two diverged copies had *each* grown independent improvements; I preserved both rather than treating one as canonical and clobbering the other.
- **Controlled fixtures for hard-to-trigger paths.** Tested incremental/liveness behavior with a temp file I grew and rotated by hand + a monkeypatched path lookup — exercising cold-load / append / partial-line / rotation deterministically without touching real data.
- **Asked when genuinely ambiguous.** A one-word instruction that would have *overridden the user's own just-made edit* → I asked a focused question with code previews rather than guessing.

## What Went Wrong

- **A task list went stale.** I opened a progress list for multi-file work, updated it twice, then let it lapse across later steps. Either maintain it or don't open one — a stale list is noise.
- **Expanded past the stated "step" boundary.** When work was framed as discrete steps, one step required touching more than its nominal scope to keep things working; I did it (and flagged it), but confirming the boundary first would have been cleaner.
- **Verbosity.** Several summary tables per turn — tolerated, but real cost.

## Lessons Learned

1. **Probe the runtime, don't theorize.** For "how does X identify/map to Y" in a live system, inspect process environment / parent chain / open file descriptors first. Systems often expose identity via environment variables that collapse a hard external-inference problem into a trivial internal read.
2. **Reproduce the exact failing path before attributing a break.** A "broke" report can be environmental or temporal (data aged out of a time window), not your recent change.
3. **A shared global cache that evicts is incompatible with a "complete per-entity index" requirement.** Key by the entity, isolate its maps under a per-entity scope, and evict whole entities at once — lookups become leak-proof and complete, and derived structures stop depending on eviction or prior-run state.
4. **Tail a live append-only file by only advancing the byte offset past complete newline-terminated lines.** Leave a mid-write partial line unconsumed so it's re-read whole on the next pass.
5. **Keep filtering at the right layer.** A "hide these types" filter belongs at the render/view stage, not the parse stage — pushing it into parse silently *severed* structural links that routed through the hidden items. Parse complete; filter the view.
6. **State confidence explicitly** — separate "reproduced" from "inferred by reading the code."

## Tips & Tricks

- Many agent/CLI runtimes export their own identity (session id, executable path, an "am I running under it" flag) into every child process's environment. Grepping the environment for the product name reveals what's available for authoritative self-identification — far better than guessing from file mtimes.
- To test file-tailing/liveness deterministically: monkeypatch a module-level path lookup and write to a temp file you grow/rotate by hand.
