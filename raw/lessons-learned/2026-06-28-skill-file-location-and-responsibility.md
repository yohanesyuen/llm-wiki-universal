## What Went Wrong

- **Edited the wrong skill file**: when asked to update the karpathy-llm-wiki SKILL.md, I searched for files mentioning "karpathy" and found a marketplace copy at `.claude/plugins/marketplaces/...`. I edited that instead of the active skill at `~/.claude/skills/karpathy-llm-wiki/SKILL.md`. The active skill file is always identified by the "Base directory for this skill:" line in the skill invocation message.
- **Reflect did the ingest itself**: during the earlier reflect, I manually compiled the wiki article and updated index.md and log.md instead of delegating to `/karpathy-llm-wiki`. The reflect skill's job is to write to `raw/` only.
- **Confused about rejected tool calls**: after the user rejected an edit, I tried to build on top of the not-yet-applied change, causing an edit mismatch. A rejected tool call leaves the file exactly as it was.

## Lessons Learned

1. **Skill base directory is authoritative**: the invocation header always shows "Base directory for this skill: <path>". The SKILL.md to edit is `<path>/SKILL.md`. Never search for skill files by keyword — you'll find stale marketplace copies.

2. **Responsibility belongs in the skill that does the work**: when skill A delegates to skill B, the format/behavior spec (e.g. log entry format) belongs in skill B's SKILL.md, not A's. A should only say "invoke B" — B owns how it behaves.

3. **Rejected tool calls don't change file state**: if a user rejects an Edit or Write, the file is unchanged. Don't attempt to build on top of an edit you think was applied but wasn't — re-read the current state first.

4. **Reflect scope is raw/ only**: the reflect skill's job ends at writing `raw/lessons-learned/YYYY-MM-DD-slug.md`. The ingest (compile article, update index, append log) is karpathy-llm-wiki's responsibility entirely.

## Tips & Tricks for Claude Code

- **Tip**: Before editing any skill's SKILL.md, look for the "Base directory for this skill:" in the most recent invocation of that skill in the conversation. That path is ground truth.
- **Tip**: When a tool call is rejected, treat the file as completely unmodified — don't infer partial application or chain edits on top of it.
