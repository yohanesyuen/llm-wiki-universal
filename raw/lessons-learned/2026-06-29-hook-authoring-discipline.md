# Session Reflection: Hook Authoring Discipline

**Date**: 2026-06-29
**Session Goal**: Wire a PreToolUse wiki-index-lookup hook and a Stop corrections-detection hook into Claude Code settings.

---

## What Went Well

- Pipe-tested the hook script before wiring it into settings — zero integration bugs.
- When moving a hook between event types, correctly updated both the settings event key and the `hookEventName` field in the script output, and caught the stdin payload shape change in the same pass.
- Validated settings JSON with `jq -e` after each edit.

## What Went Wrong

- **Hardcoded home directory**: First draft embedded an absolute path to the home directory — required correction. Hook scripts are reusable artifacts; portable path resolution must be used from the start.
- **Wrong implementation language**: Wrote the hook in bash when the user wanted Python. Should check stated preferences before choosing a language.
- **Memory scoped to CWD**: Saved a feedback lesson to the project-derived memory path (which only loads when that directory is the CWD) instead of the user-scoped memory directory. Generic lessons need the user-scoped location.
- **Bypassed wiki ingest**: Wrote the feedback item straight to memory rather than deferring to the reflect/ingest flow. Generic lessons belong in the wiki, not in session-scoped memory.

## Lessons Learned

1. **Portable paths in hook scripts**: Use language-native home resolution (`Path.home()` in Python, `$HOME` or `~` in shell). Never embed an absolute path containing a username. Apply on first draft.

2. **Auto-memory is CWD-scoped by default**: The default memory path is derived from the working directory and only loads in that context. Generic, session-independent lessons need an explicitly configured user-scoped memory directory.

3. **Hook event migration is a three-part change**: Moving a hook between events requires updating (a) the event key in settings, (b) the `hookEventName` in the script output, and (c) any stdin field references — payload shape differs per event.

4. **Generic corrections belong in the wiki, not memory**: When a user correction reveals a reusable rule, the right destination is the wiki via the reflect → ingest flow. Memory is for mid-session recall; the wiki is for durable cross-session lessons.

## Action Items

- [ ] On any hook script first draft: scan for hardcoded absolute paths before writing.
- [ ] Before choosing a hook implementation language: check user's stated preference from memory.
- [ ] Before saving to memory: ask — generic rule (wiki) or session-specific fact (memory)?
