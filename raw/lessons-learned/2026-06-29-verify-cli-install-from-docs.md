# Session Reflection: Verify CLI Install Commands from Official Docs

**Date**: 2026-06-29
**Session Goal**: Add Spec Kit scaffolding to git and update CLAUDE.md with the correct install requirement and auto-install snippet.

---

## What Went Well

- Fetching the README via `ctx_fetch_and_index` immediately after the edit rejection gave a clean, authoritative answer in one round-trip
- Batching `ctx_search` with multiple queries extracted both the install command and the CLI name in a single call
- Committing all 37 files (`.specify/`, skills, CLAUDE.md) in a single atomic commit was clean

## What Went Wrong

- First CLAUDE.md edit used `npm install -g @github/spec-kit` — a fabricated install command that assumed npm without any evidence; user rejected it and had to prompt me to read the README
- The CLI name was also wrong (`spec-kit` instead of `specify`); both the package manager and the binary name were hallucinated from training priors

## Lessons Learned

1. **Never write an install command from memory for any tool**: Install commands (package manager, registry name, binary name, flags) are highly specific and change between releases. Always fetch the official README or install guide before writing one — even for well-known tools.

2. **A rejected edit with "check the docs" means the error is factual, not stylistic**: When an edit is rejected with a pointer to documentation, the fix is to read that documentation first, not to rephrase the guess. The `ctx_fetch_and_index` + `ctx_search` pattern resolves this in two tool calls.

3. **Package manager ≠ language assumption**: A tool living in a GitHub repo does not default to npm. Spec Kit uses `uv` (a Python-ecosystem tool manager) despite being a general-purpose CLI. Verify the ecosystem from the README prerequisites section.

## Action Items

- [ ] Before writing any install snippet, run `ctx_fetch_and_index` on the tool's README and search for "install" — never guess
- [ ] When an edit is rejected with a doc pointer, treat it as a research task, not a rewrite task

## Tips & Tricks for Claude Code

- **`ctx_fetch_and_index` + `ctx_search` is the correct pattern for "check the README"**: fetch the raw URL, then search for install/prerequisites/usage — keeps raw bytes out of context
- **Edit rejections with doc pointers are hard stops**: don't re-attempt with a modified guess; fetch the source and derive from that
