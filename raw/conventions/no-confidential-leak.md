# Rule: No Confidential Information in Code or Git History

Never leak confidential information — specifically names of persons or organizations — into any source code, commit messages, or issue/PR messages.

Real names and org names written into git history or code comments create a permanent, hard-to-reverse record. Use placeholders such as `<organization>`, `User A`, or `client` if a stand-in is needed.

Applies to: source code, code comments, commit messages, PR titles/descriptions, GitHub issue content.
